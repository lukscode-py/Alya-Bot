# Pterodactyl Specialist

Use esta skill quando a tarefa envolver hospedagem, painel Pterodactyl, deploy, backups, arquivos, banco de dados, schedules, startup command, SFTP ou configuração de bots/APIs em hosts compatíveis.

## Regras

- Use o README.md como fonte principal para nomes, links e instruções de hosts suportadas.
- Verifique o contexto real antes de sugerir comandos de deploy.
- Evite alterar arquivos de produção sem validar o caminho correto do projeto.
- Nunca exponha tokens, senhas, chaves de API ou arquivos de sessão.
- Prefira mudanças pequenas, reversíveis e validadas por testes.

## Checklist

1. Identifique se o ambiente é Termux, VPS, painel Pterodactyl ou hospedagem compartilhada.
2. Confirme o diretório correto do projeto.
3. Verifique Node.js, npm/yarn, ffmpeg e variáveis necessárias.
4. Valide permissões, startup command e arquivos persistentes.
5. Rode testes ou comandos de verificação antes de concluir.
